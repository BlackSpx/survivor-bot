----- HOW TO APPLY ON RAILWAY (step by step) -----

Note: change nothing and the bot still redeploys fine + the linked-only chat rule
works automatically. Below is only to switch ON the new optional features.

  1. Push the updated code to GitHub → Railway auto-redeploys.
  2. Make sure the volume + DATABASE_PATH=/data/survivor.db are set (so the auto
     DB migration for voice_seconds persists, and points aren't wiped).
  3. (Optional) Railway → service → Variables, add only what you want:
       - GAME_VOICE_CHANNEL_ID=<voice channel id>   (turns ON voice tracking)
       - TRACK_ALL_GAMES=true                        (count non-Forest games)
       - VOICE_KICK_UNLINKED=true                    (kick unlinked voice joiners)
       - VOICE_LINK_GRACE_SECONDS=120                (optional, default 120)
  4. Discord → Server Settings → Roles → the bot's "Survivor" role, enable:
       - Manage Messages  (needed for linked-only chat deletion)
       - Move Members     (only if VOICE_KICK_UNLINKED=true)
     (No re-invite needed. If the voice channel is private, also give View Channel.)
  5. Intents: nothing to do — GuildVoiceStates is NOT a privileged intent (no
     Developer Portal toggle). Keep Server Members + Message Content as-is.
  6. Confirm in Railway Deploy Logs:
       "⚙️  ... | all-games: on"
       "🎙️  Voice tracking on for channel <id> (kick unlinked: yes, after 120s)."
     (The voice line only shows if GAME_VOICE_CHANNEL_ID is set.)

  Minimum to use the new stuff: set GAME_VOICE_CHANNEL_ID (+ optionally
  TRACK_ALL_GAMES / VOICE_KICK_UNLINKED) and give the role Manage Messages
  (+ Move Members if kicking). Everything else is automatic.
